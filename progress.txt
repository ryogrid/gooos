## Codebase Patterns
- Use `//go:linkname` with bodyless Go function declarations to call assembly stubs from Go code
- outb/inb stubs in stubs.S follow SysV AMD64 ABI (rdi=first arg, rsi=second arg, rax=return)
- QEMU direct `-kernel` boot fails for ELF64; use ISO boot via `make iso` + `make run` instead
- Serial output uses COM1 (0x3F8) at 115200 baud 8N1, verified with `-serial stdio`
- TinyGo builds the entire `./src` package, so new `.go` files are automatically included
