## Codebase Patterns
- Use `//go:linkname` with bodyless Go function declarations to call assembly stubs from Go code
- outb/inb stubs in stubs.S follow SysV AMD64 ABI (rdi=first arg, rsi=second arg, rax=return)
- QEMU direct `-kernel` boot fails for ELF64; use ISO boot via `make iso` + `make run` instead
- Serial output uses COM1 (0x3F8) at 115200 baud 8N1, verified with `-serial stdio`
- TinyGo builds the entire `./src` package, so new `.go` files are automatically included
- IDT descriptor for lidt must be packed (2-byte limit + 8-byte base = 10 bytes); use [10]byte array in Go since structs get alignment padding
- To get an assembly function's address in Go, create a helper that returns it via `leaq symbol(%rip), %rax`
- GDT64_CODE selector = 0x08 (second GDT entry at offset 8)
- ISR stubs use ISR_NOERR/ISR_ERR macros; vectors with CPU error code: 8, 10-14, 17, 21
- .altmacro + .rept + %COUNTER pattern generates bulk assembly stubs (used for vectors 32-255 and the ISR address table)
- ISR address table in .rodata: array of 256 .quad pointers, accessed from Go via isrTableAddr()
- go_interrupt_handler is exported with //export for C-ABI calling from assembly
- Stack alignment before Go call: save RSP in callee-saved register, andq $-16 to align, restore after call
- PIC remap: master PIC 0x20/0x21, slave PIC 0xA0/0xA1; ICW1-ICW4 sequence; master offset 32, slave offset 40
- Always register default IRQ handlers (32-47) that send EOI to prevent PIC hanging on spurious interrupts
- After sti, add hlt loop at end of main() to keep kernel alive while handling interrupts
- PIT channel 0: command 0x34 (channel 0, lobyte/hibyte, rate generator mode 2, binary); divisor = 1193182/100 = 11932
- Register PIT handler AFTER default IRQ handlers, so it overrides the default for vector 32
- PS/2 keyboard: read scancode from port 0x60 in IRQ1 handler; bit 7 set = key release (ignore); scancode set 1 for make codes
- VGA echo buffer pattern: maintain a byte array + length, redraw entire line on change, clear trailing chars with vgaClearLine
- go_interrupt_handler accepts (vector, errorCode); error code stored in lastErrorCode global for handlers to read
- CR2/CR3/invlpg stubs in stubs.S; readCR3() &^ 0xFFF gets PML4 base; identity mapping means phys=virt for page tables
- Page frame allocator: bump allocator starting at _heap_end aligned to 4 KiB; heapEndAddr() stub returns linker symbol value
- mapPage walks 4-level page table (PML4→PDP→PD→PT); does NOT split existing 2 MiB huge pages
- For VM demo, use virtual addresses outside the boot-time 1 GiB identity map (e.g., 0x40000000) to avoid huge page conflicts
- Context switch: save/restore callee-saved regs (rbx, rbp, r12-r15) + RSP; new task stacks need zeroed regs + entry address + halt safety net
- New tasks start from inside a timer ISR (IF=0); must call sti() first to re-enable interrupts
- Assembly address helpers (leaq symbol(%rip), %rax; ret) get //export Go function addresses for task entry points
- Schedule from timer handler: EOI first, then switchContext; interrupt gates keep IF=0 during the switch
- isr_common's %rbx (pre-alignment RSP) is preserved across context switches because switchContext saves/restores callee-saved regs
- GDT can be rebuilt in Go at runtime: define [N]uint64 array, pack TSS descriptor as two entries, lgdt + far return to reload CS, ltr loads TSS
- TSS RSP0 must point to kernel stack for Ring 3→Ring 0 transitions; CPU loads RSP from TSS.RSP0 on privilege change
- User-accessible pages need pageUser (bit 2) set at ALL page table levels (PML4E, PDPE, PDE, PTE); walkOrCreate propagates User flag
- IDT gate DPL=3 (TypeAttr=0xEE) required for int 0x80 from Ring 3; DPL=0 gates cause #GP from user mode
- iretq to Ring 3: push SS(0x23), RSP, RFLAGS, CS(0x1B), RIP; user CS = 0x18|RPL=3, user SS = 0x20|RPL=3
- User code pages mapped at vaddr > 1 GiB (outside boot identity map) with pagePresent|pageWrite|pageUser flags
- After jumpToRing3, task 0 runs in Ring 3; timer preempts via TSS RSP0 kernel stack; scheduler context-switches normally
- In-memory FS: global `fs` FileSystem with flat [16]FileEntry array; each entry has fixed [4096]byte data, name string, size int, used bool
- VGA line assignments after US-011: serial 0, IDT 1, ISR 2, PIT 3, kbd 4, ints 5, GC 6-8, VM 9, FS 10, timer 11, GDT 12, kbd-echo 12, sched 13-14, tasks 15-17, user 18, SMP 19
- AP trampoline lives in .rodata, copied to 0x8000 at runtime; uses hardcoded absolute addresses assuming TRAMP_BASE=0x8000
- LAPIC at 0xFEE00000 must be identity-mapped via mapPage (outside boot 1 GiB range); use pagePCD|pagePWT for MMIO
- ACPI RSDP found by scanning BIOS ROM area 0xE0000-0xFFFFF (16-byte aligned); RSDT→MADT gives AP count
- INIT-SIPI-SIPI: INIT IPI (ICR=0xC4500), wait 10ms, SIPI×2 (ICR=0xC4608 for vector 0x08=0x8000/0x1000)
- APs share BSP's PML4 (same CR3); trampoline GDT has null+code32+data+code64 entries for mode transitions
- AP entry function must avoid heap allocation (GC not thread-safe); use serialPutChar for output
- lock xaddl atomically claims AP index from shared counter; each AP loads its stack from a pre-indexed array
- After LAPIC enable (SVR bit 8), configure LVT LINT0=ExtINT to preserve PIC interrupt delivery to BSP
